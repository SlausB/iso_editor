function Dispatcher.ActionsCount (   queueId:int)

Какое количество действий находится внутри указанной очереди.

Аргументы:
queueIdИдентификатор очереди, количество действий внутри которой необходимо определить.
Возвращает:
Количество действий в указанной очереди. Возвращает ноль если очереди с таким идентификатором не существует.
function Dispatcher.AddAction (   queueId:int,
  actionDesc:ActionDesc 
) [private]

Добавление нового действия в указанную очередь.

Указанное действие добавляется в конец очереди.
Если номер очереди == -1, то к списку очередей будет добавлена новая очередь и действие добавится в неё.

Аргументы:
queueIdИдентификатор очереди, в конец которой будет добавлено действие.
actionПервый элемент - номер действия. Остальные - параметры.
Возвращает:
Идентификатор очереди, в которую было добавлено действие.
function Dispatcher.AppendRedirection (   id:int,
  queue:Queue 
) [private]

Добавление новой очереди.

Если id указан неверно, то очередь добавлена не будет, поэтому, чтобы не было утечек памяти, queue нужно удалить извне (везёт если в языке программирования не бывает утечек памяти).

Аргументы:
idИдентификатор добавляемой очереди.
queueУказатель на очередь, которую нужно добавить.
Возвращает:
Идентификатор добавленной очереди - отличается от id только если id == -1. -1 - очередь НЕ была добавлена: неверный id .

function Dispatcher.Clear (   queueId:int = -1,
  resetIgnoring:Boolean = false,
  exceptions 
)

Удаление очереди с указанным идентификатором.

По-умолчанию будут удалены все очереди. Если указать какую-то конкретную очередь, то очередь будет очищена.
Исключения удаления очередей влияют на удаление только если указано удалить все очереди (первый параметр = -1, что по-умолчанию).

Аргументы:
queueIdИдентификатор очереди, которую нужно очистить. По-умолчанию будут удаляться все очереди.
resetIgnoringtrue - игнорирование очереди будет отменено. По-умолчанию = false.
exceptionsПеречисленные очереди, которые НЕ нужно удалять.

function Dispatcher.CreateAction (   queueId:int,
  actionDesc:ActionDesc 
)

Создание и добавление нового действия в указанную очередь.

Аргументы:
queueIdИдентификатор очереди, в которую нужно добавить очередь. Идентификатор может быть любым числом >= 0. Если -1, то будет создана очередь с первым найденным свободным идентификатором и действие будет добавлено в неё. Действия будут выполняться в порядке возрастания идентификаторов.
actionDescОписание добавляемого действия.
Возвращает:
Идентификатор очереди, в которую было добавлено действие.

function Dispatcher.IgnoreQueue (   queueId:int)

Игнорирование указанной очереди.

Состояние действий в очереди останется прежним, но они перестанут выполняться.
После отмены игнорирования выполнения продолжиться с остановленного места.

Аргументы:
queueIdИдентификатор очереди, которая должна игнорироваться. Если == -1, то будут игнорироваться все очереди.
function Dispatcher.IgnoreThread (   thread:int = -1)

Игнорирование указанного потока.

Если поток проигнорирован, то все вызовы функции Live() с этим потоком будут игнорироваться.
Идентификаторы потоков могут быть только положительными.

Аргументы:
threadПоток, который нужно игнорировать. Если = -1, что по-умолчанию, то будут игнорироваться все потоки.

function Dispatcher.Live (   elapsedSeconds:Number = 0.020,
  thread:int = 0 
)

Выполнение текущих действий в очередях.

Аргументы:
elapsedSecondsСколько секунд прошло с момента прошлого вызова Live() . Это время, на которое текущие действия (в разных очередях) могут продвинуться в выполнении.
threadИдентификатор потока в котором вызывается функция.
Каждый определённый поток можно игнорировать функцией Dispatcher.IgnoreThread() .
По умолчанию = 0. Идентификаторы потоков могут быть только положительными.
